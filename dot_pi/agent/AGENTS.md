# Global Agent Instructions

## Commit Message Format

- Use Conventional Commits.
- Do not use scopes in commit types (e.g., use `chore: ...`, not `chore(chezmoi): ...`).

## Commit Message Best Practices

- Keep the commit title around 50 characters.
- Add a blank line between title and body.
- Wrap body text at around 72 characters per line.
- Multiple paragraphs are allowed in the body (separate them with blank lines).
- In the body, explain **why** the change was made, not **what** changed.
